nl2sql

Natural language to SQL via a single agent loop.

Overview

nl2sql turns natural-language questions into validated SQL queries against a real database. It runs an agent loop with the four core tools the Microsoft ISE article identifies as load-bearing: list tables, full schema, per-table schema, and execute SQL. On top of that it adds two of the article's highest-leverage capabilities: a domain hints system (+10 to 14 points accuracy) and a feedback store for corrected SQL (+19 points).

  • Safety-first SELECT-only by default, single statement, system-table denylist.
  • Pluggable LLM and DB behind clean ABCs - Anthropic, OpenAI, Postgres, SQLite ship in v1.
  • CLI & library Equal first-class status; the CLI is built on Typer + Rich.
  • Eval harness with HTML report, numeric tolerance, bipartite row matching.

Install

pip install nl2sql                  # core
pip install "nl2sql[anthropic]"     # + Anthropic
pip install "nl2sql[openai]"        # + OpenAI
pip install "nl2sql[postgres]"      # + Postgres driver
pip install "nl2sql[all]"           # everything

Quickstart

from nl2sql import Nl2Sql
from nl2sql.db import SqliteDatabase
from nl2sql.llm import AnthropicClient

n2s = Nl2Sql(
    db=SqliteDatabase("./customers.db"),
    llm=AnthropicClient(model="claude-opus-4-7"),
)

result = n2s.ask("How many customers signed up last quarter?")
print(result.sql)
print(result.rows)

With hints and feedback

from nl2sql import Nl2Sql, DomainHints, JsonFeedbackStore
from nl2sql.db import PostgresDatabase
from nl2sql.llm import OpenAIClient

n2s = Nl2Sql(
    db=PostgresDatabase(dsn="postgresql://reader@host/analytics"),
    llm=OpenAIClient(model="gpt-5"),
    hints=DomainHints.from_yaml("hints.yaml"),
    feedback=JsonFeedbackStore("./feedback.json"),
    max_iterations=12,
)

result = n2s.ask("Top 10 customers by 2026 spend, excluding internal accounts")
n2s.feedback.record(
    question=result.question,
    sql=result.sql,
    correct=True,
    notes="internal accounts are flagged by accounts.is_internal",
)

CLI

The CLI is built on Typer and Rich. Every command supports --json, --profile, --db, --model, --quiet, --no-color.

nl2sql init                              # scaffold .nl2sql.yaml + hints.yaml
nl2sql ask "How many orders this month?"
nl2sql repl                              # interactive REPL
nl2sql inspect tables
nl2sql inspect schema --table households
nl2sql inspect sample customers -n 5
nl2sql eval run ./datasets/livesqlbench-medium --output ./reports/run-001
nl2sql eval show ./reports/run-001 --failures-only
nl2sql feedback list
nl2sql feedback review
nl2sql feedback forget <id>
nl2sql hints validate ./hints.yaml
nl2sql hints show
nl2sql version

Configuration

Resolved in this order, later overriding earlier:

  1. ~/.config/nl2sql/config.yaml - user defaults
  2. ./.nl2sql.yaml - project config (or any ancestor)
  3. --profile <name> selects a named profile
  4. Env vars: NL2SQL_DB_URL, NL2SQL_LLM_MODEL, ANTHROPIC_API_KEY, OPENAI_API_KEY
  5. CLI flags
default_profile: dev

profiles:
  dev:
    db: sqlite:///./customers.db
    llm:
      provider: anthropic
      model: claude-opus-4-7
    hints: ./hints.yaml
    feedback: ./feedback.json
    max_iterations: 10

  prod:
    db: postgresql://reader@db.internal/analytics
    llm:
      provider: openai
      model: gpt-5
    hints: ./hints.prod.yaml
    feedback: ./prod.feedback.json
    require_select_only: true

Nl2Sql

Nl2Sql(
    db: Database,
    llm: LLMClient,
    *,
    hints: DomainHints | None = None,
    knowledge_store: KnowledgeStore | None = None,
    feedback: FeedbackStore | None = None,
    max_iterations: int = 10,
    max_rows_returned: int = 200,
    query_timeout_s: float = 10.0,
    require_select_only: bool = True,
    deny_system_tables: bool = True,
)

Calling .ask(question) runs the agent loop and returns a Result with the SQL the agent ran, the rows it returned, the full step-by-step trace, token usage, and a stopped_reason tag.

Database

nl2sql.db ships SqliteDatabase and PostgresDatabase. Roll your own by subclassing Database and implementing list_tables, get_schema, get_table_schema, and execute_select.

LLMClient

Built-ins: AnthropicClient, OpenAIClient, MockLLMClient (for tests; replays scripted tool calls deterministically). Implement the chat method to plug in any other provider.

Domain hints

Static hints inject directly into the system prompt under ## Glossary, ## Formulas, ## Join rules, ## Column descriptions. Empty fields render no headings - clean prompt by default.

DomainHints(
    glossary={"locregion": "Region code, uppercase, no diacritics"},
    formulas={"warranty_claim_rate": "returns_with_warranty / total_returns * 100"},
    join_rules=["households.housenum links to properties.houselink"],
    column_descriptions={"households.locregion": "Region code"},
)

KnowledgeStore exposes a lookup_hint(topic) tool to the agent. Implement lookup(topic) -> str | None to plug in vector search or any other store. DictKnowledgeStore ships out of the box.

Feedback store

JsonFeedbackStore is file-backed, with token-Jaccard similarity for retrieval. Records are read on every agent run; the top-k matches are appended to the system prompt under ## Past validated examples.

store = JsonFeedbackStore("./feedback.json")
store.record(question, sql, correct=True, notes="customer tier is accounts.tier")
store.find_similar("Top customers by spend", k=3)
store.forget(entry_id)

Eval harness

from nl2sql.eval import run_eval, FlexibleMatcher, LiveSQLBenchDataset

ds = LiveSQLBenchDataset.from_path("./bench/cases.jsonl")
report = run_eval(
    dataset=ds,
    system=n2s,
    matcher=FlexibleMatcher(numeric_tolerance=0.01, row_matching="bipartite"),
    parallel=4,
)
report.write_html("./reports/run-001/report.html")
report.write_json("./reports/run-001/report.json")
print(report.accuracy)

Each case can specify expected_rows (preferred, scored with FlexibleMatcher) or expected_sql (whitespace-normalised exact match fallback).

Safety

Three hard rails live in nl2sql.safety:

  1. Single-statement check (parsed via sqlglot, not split on semicolons).
  2. SELECT-only allowlist; CTEs and unions are allowed; INSERT/UPDATE/DELETE/DDL are not.
  3. System-table denylist: pg_*, information_schema, sqlite_master, sqlite_sequence. Disable with deny_system_tables=False.
from nl2sql.safety import is_safe_select, enforce_safe_select

is_safe_select("SELECT 1").ok       # True
is_safe_select("DROP TABLE t").ok   # False
enforce_safe_select("SELECT 1")     # returns normalised SQL
enforce_safe_select("DELETE FROM t") # raises SafetyError

Known limits

  • Business-logic errors require domain knowledge that no amount of prompt tuning fixes; surface them via hints + feedback.
  • Schema dumps are flat strings injected into the prompt; databases with thousands of tables will blow context. v2 will add vector-indexed retrieval.
  • No streaming of intermediate steps in v1. Callers receive the final Result.
  • Single-database scope per Nl2Sql instance.
